设备指纹的核心目标是 在不依赖用户授权的前提下，生成`稳定、唯一、难篡改`的设备标识，同时兼顾兼容性（支持 IE11+ 及现代浏览器）和隐私合规性。
# 核心特征选择（按稳定性优先级排序）
|特征类别|具体特征|作用|兼容性|
|--|--|--|--|
|浏览器核心特征|User-Agent（UA）|区分浏览器类型、版本、系统|所有浏览器支持|
|浏览器语言（navigator.language）|补充系统 / 区域特征|所有浏览器支持|所有浏览器支持|
|时区偏移（getTimezoneOffset）|系统时区特征（稳定）|所有浏览器支持|所有浏览器支持|
|屏幕分辨率（screen.width/height）|设备硬件特征（稳定）|所有浏览器支持|所有浏览器支持|
|颜色深度（screen.colorDepth）|显示设备特征|所有浏览器支持|所有浏览器支持|
|浏览器功能支持|支持的 API 列表（如 WebGL、Canvas）|浏览器能力特征（差异大、稳定性高）|IE11+ 支持核心 API|
|Cookie 启用状态（navigator.cookieEnabled）|浏览器配置特征|所有浏览器支持|所有浏览器支持|
|Canvas 指纹（核心）|Canvas 绘图哈希|利用 GPU 渲染差异，生成设备唯一特征|IE11+ 支持|
|WebGL 指纹（补充）|WebGL 渲染器 / 供应商信息|显卡硬件特征（稳定性极高）|IE11+ 支持|
|存储特征（辅助）|localStorage 支持状态|浏览器存储能力特征|IE8+ 支持|

## 说明
- Canvas 指纹：通过在 Canvas 绘制固定图形（文字、矩形），再将画布数据转为 Base64 后哈希。不同设备的 GPU 渲染、字体渲染存在细微差异，导致最终哈希值不同，是设备指纹的核心特征；
- WebGL 指纹：获取显卡渲染器（WEBGL_debug_renderer_info）和供应商信息，硬件相关特征稳定性远高于软件特征；
- 特征去重：部分特征可能存在冗余（如 UA 已包含系统信息），但多特征组合可降低重复率。
