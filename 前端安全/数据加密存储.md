# 高安全业务（如本地存储敏感数据：收货地址、支付信息）

## 双重密钥分层：
- 后端生成「主密钥」（会话级）+「数据密钥」（每次存储生成）；
- 主密钥：会话级，内存保存，用于加密「数据密钥」；
- 数据密钥：每次存储数据时随机生成，加密数据后，用主密钥加密数据密钥，再和密文一起存入本地；
- 优势：即使主密钥泄露，旧数据的「数据密钥」若已销毁，依然无法解密。

## 定期主动轮换：
- 后端每 30 分钟自动为活跃会话生成新密钥，前端通过心跳接口获取新密钥，平滑替换内存中的旧密钥；
- 旧密钥解密后的数据，用新密钥重新加密存储，实现密钥无感知轮换。

# 关键落地细节（避坑）
- 密钥传输安全：
  - 密钥从后端返回前端时，必须通过 HTTPS 传输，且建议对密钥本身再做一层轻量加密（比如用登录 Token 的前 32 位做临时密钥加密）；
- 密钥失效机制：
  - 后端在 Redis 中存储「登录 Token - 密钥」映射，Token 失效（过期 / 登出）时，立即删除该映射，前端再请求密钥会返回失败，触发重新登录；
- 本地数据兼容：
  - 密钥轮换后，前端需先用旧密钥解密本地数据，再用新密钥重新加密存储，避免数据丢失；
  - 解密失败时（如密钥失效），直接清空该条本地数据，从服务端重新拉取。
