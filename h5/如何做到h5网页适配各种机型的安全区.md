# 一、安全区适配的核心原则（先明确方向）

- 优先 CSS 原生方案：用 env()/constant() + viewport-fit=cover，避免过度依赖 JS 计算（性能更高、兼容性更好）；
- 最小侵入性：仅对「固定定位 / 全屏元素」做适配，普通文档流元素由浏览器自动处理；
- 兼容兜底：同时支持新旧 iOS 版本，非 iOS 设备无感知降级；
- 统一管理：用 CSS 变量集中管理安全区数值，便于维护；
- 多容器适配：兼顾 Safari / 微信 WebView/App 内嵌 WebView 等场景。

# 二、标准化落地步骤

## 步骤 1：基础配置（全局唯一，一次配置）

### 1.1 视口配置（必须加 viewport-fit=cover）
```html
<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, minimum-scale=1.0, user-scalable=no, viewport-fit=cover">
```
- 禁止用户缩放（user-scalable=no）：避免缩放破坏安全区计算；
- viewport-fit=cover：核心，让页面撑满全屏，触发安全区变量计算。

### 1.2 CSS 变量定义（统一管理安全区数值）
```css
/* 根节点定义安全区变量，兼容新旧 iOS */
:root {
  /* 旧 iOS（11.0-11.2）：constant + 兜底值 */
  --safe-top: constant(safe-area-inset-top, 0px);
  --safe-bottom: constant(safe-area-inset-bottom, 0px);
  --safe-left: constant(safe-area-inset-left, 0px);
  --safe-right: constant(safe-area-inset-right, 0px);
  /* 新 iOS（11.2+）：env + 兜底值，覆盖上面 */
  --safe-top: env(safe-area-inset-top, 0px);
  --safe-bottom: env(safe-area-inset-bottom, 0px);
  --safe-left: env(safe-area-inset-left, 0px);
  --safe-right: env(safe-area-inset-right, 0px);
}

/* 基础重置：消除默认样式干扰 */
* {
  margin: 0;
  padding: 0;
  box-sizing: border-box;
}

/* 强制触发 Safari 安全区计算（解决 Safari 数值为 0 的问题） */
html {
  height: 100%;
}
body {
  min-height: 100vh; /* 关键：让 Safari 识别为全屏页面 */
  /* 页面整体安全区：避免内容被刘海/小黑条遮挡 */
  padding-top: var(--safe-top);
  padding-bottom: var(--safe-bottom);
  /* 横屏时左右安全区适配 */
  padding-left: var(--safe-left);
  padding-right: var(--safe-right);
}
```
## 步骤 2：场景化适配（按元素类型处理）

### 2.1 底部固定元素（tabbar / 按钮 / 弹窗）

```css
/* 核心：padding-bottom 预留安全区，box-sizing: content-box 避免挤压内容 */
.fixed-bottom {
  position: fixed;
  bottom: 0;
  left: 0;
  right: 0;
  height: 50px; /* 基础高度 */
  padding-bottom: var(--safe-bottom);
  box-sizing: content-box; /* 关键：内边距不占用高度 */
}

/* 示例：底部按钮 */
.bottom-btn {
  position: fixed;
  left: 20px;
  right: 20px;
  bottom: var(--safe-bottom); /* 直接偏移安全区，替代 padding */
  height: 50px;
  border-radius: 8px;
  background: #007aff;
  color: white;
  border: none;
}
```
### 2.2 顶部固定元素（导航栏 / 状态栏）

```css
.fixed-top {
  position: fixed;
  top: 0;
  left: 0;
  right: 0;
  height: 44px; /* 基础高度 */
  padding-top: var(--safe-top);
  box-sizing: content-box;
  /* 可选：适配状态栏文字颜色 */
  color: white;
  background: #007aff;
}
```
### 2.3 全屏容器（弹窗 / 浮层）

## 步骤 3：多容器兜底适配（解决特殊场景）

### 3.1 Safari 专属兜底（数值为 0 时）

```js
// 页面加载后检测，若安全区数值为 0，手动设置（适配 iOS 16+ Safari）
window.addEventListener('load', () => {
  const getCssVar = (name) => getComputedStyle(document.documentElement).getPropertyValue(name).trim();
  const safeBottom = getCssVar('--safe-bottom');
  
  // 若 Safari 中安全区为 0，且是 iOS 全面屏，手动设置默认值
  if (safeBottom === '0px' && /iPhone/.test(navigator.userAgent) && screen.height / screen.width > 1.8) {
    document.documentElement.style.setProperty('--safe-bottom', '34px');
    document.documentElement.style.setProperty('--safe-top', '59px');
  }
});
```

### 3.2 App 内嵌 WebView 适配（原生协作）
- 原生端：计算安全区并通过 URL 参数 / JS 桥传递；
- H5 端：接收参数并覆盖 CSS 变量（优先级最高）：
```js
const urlParams = new URLSearchParams(window.location.search);
const safeBottom = urlParams.get('safeBottom');
if (safeBottom) {
  document.documentElement.style.setProperty('--safe-bottom', `${safeBottom}px`);
}
```
