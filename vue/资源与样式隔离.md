# 资源与样式隔离：scoped 的原理与局限，CSS Modules 的取舍

## scoped 样式：核心原理与特性

scoped 的本质是CSS 作用域隔离，通过「属性选择器 + 哈希值」实现样式仅作用于当前组件.

- 编译阶段：Vue 会给组件的根元素及所有子元素添加一个唯一的 data-v-xxx 属性（xxx 是随机哈希值）。
- 样式转换：给组件内的所有 CSS 选择器末尾追加 [data-v-xxx] 属性选择器，让样式仅匹配当前组件的元素。

示例：编译前后对比

```vue
<!-- 编写时 -->
<template>
  <div class="box">Hello</div>
</template>

<style scoped>
.box {
  color: red;
}
</style>

<!-- 编译后（浏览器渲染） -->
<template>
  <div class="box" data-v-123456>Hello</div>
</template>

<style>
.box[data-v-123456] {
  color: red;
}
</style>
```
关键特性
- 自动隔离：无需手动命名空间，样式默认只作用于当前组件，避免全局污染。
- 样式穿透：子组件的根元素会同时继承父组件的 data-v-xxx 属性（便于父组件控制子组件根样式）。
- 兼容性：基于 CSS 原生属性选择器，兼容所有现代浏览器。

## scoped 的局限性（生产中常见问题）

- 无法穿透第三方组件
  - scoped 样式无法作用于组件内部的第三方组件（如 Element Plus/Ant Design Vue 组件），因为第三方组件的内部元素没有当前组件的 data-v-xxx 属性。
- 动态生成的元素无法匹配
  - 通过 JS 动态创建的元素（如 document.createElement、v-html 渲染的 HTML），不会自动添加 data-v-xxx 属性，导致 scoped 样式失效。
- 样式权重问题
  - scoped 生成的 [data-v-xxx] 会增加选择器权重，可能导致
    - 全局样式覆盖 scoped 样式时需要更高权重（如 !important）
    - 多层嵌套的 scoped 样式权重过高，难以覆盖
- 子组件根元素的样式冲突
  - 子组件根元素会同时拥有父组件和自身的 data-v-xxx 属性，若父 / 子组件对根元素的样式命名冲突，会导致样式叠加 / 覆盖。

## CSS Modules：原理与核心优势
CSS Modules 是模块化 CSS 方案，通过「类名哈希化 + 映射表」实现样式隔离
- 编译阶段：将 CSS 文件中的类名编译为唯一的哈希值（如 .box → .box_3xt58）。
- 映射关系：生成一个 JS 对象（映射表），记录原始类名与哈希类名的对应关系。
- 组件使用：通过 $style.类名 引用哈希化后的类名，确保类名全局唯一

示例：Vue 中使用 CSS Modules
```vue
<!-- 1. 样式文件：box.module.css -->
.box {
  color: red;
}

<!-- 2. 组件中使用 -->
<template>
  <div :class="$style.box">Hello</div>
</template>

<script setup>
// 导入样式模块（必须加 .module 后缀）
import styles from './box.module.css';
console.log(styles); // { box: 'box_3xt58' }
</script>

<style module>
/* 也可直接在组件内写 module 样式，无需单独文件 */
.box {
  font-size: 16px;
}
</style>
```

核心优势
- 彻底隔离：类名哈希化，全局唯一，完全避免样式冲突（包括第三方组件、动态元素）。
- 灵活可控：可通过映射表手动控制类名，支持动态绑定、条件样式。
- 无权重问题：类名唯一，无需依赖属性选择器，样式权重清晰。
- 支持动态元素：手动给动态元素添加哈希类名即可生效。
- 类型安全：结合 TypeScript 可生成样式类型声明，避免类名拼写错误。

## scoped vs CSS Modules：取舍策略

- 优先用 scoped 的场景
  - ✅ 适合大多数常规组件：
  - 简单组件（如按钮、卡片、列表项），无复杂样式嵌套。
  - 无需深度修改第三方组件样式。
  - 团队对 CSS 模块化理解较浅，追求快速开发（scoped 无学习成本）。
  - 项目体积小，样式管理简单

- 优先用 CSS Modules 的场景
  - ✅ 适合复杂 / 核心组件：
  - 大型应用的核心组件（如表单、布局、业务模块），需彻底隔离样式。
  - 需要深度修改第三方组件样式（如自定义 Element Plus 组件的内部样式）。
  - 动态生成元素较多的场景（如富文本、图表容器）。
  - 多人协作的大型项目，需严格避免样式污染。
  - 结合 TypeScript 开发，追求类型安全。

- 混合使用的最佳实践
```vue
<template>
  <!-- 基础样式用 scoped -->
  <div class="base-box" :class="$style.custom-box">
    <el-button class="base-btn" :class="$style.custom-btn">按钮</el-button>
  </div>
</template>

<style scoped>
/* 基础样式：scoped 自动隔离 */
.base-box {
  padding: 20px;
}
/* 穿透第三方组件：用 ::v-deep（Vue3 推荐 :deep()） */
:deep(.base-btn) {
  border-radius: 4px;
}
</style>

<style module>
/* 核心样式：CSS Modules 彻底隔离 */
.custom-box {
  background: #f5f5f5;
}
.custom-btn {
  background: #409eff;
  color: white;
}
</style>
```
