# 分析方法
## 确认目标
目标需要清晰明确
## 收集数据
performance API 进行采集

## 清洗数据
去除无效值
## 统计值分析
把各种能够得到数据主表的统计值一一列举，如平均值、中位数、标准差等，观察特征。
## 时序分析
性能是随着时间变化的，在统计值分析的基础上，可以观察随着时间变化统计指标发生了什么变化。
## 维度分析
### 地域
不同地域的性能也会有较大差异，尤其是对国际化场景来说，不同国家的网络状况，CDN节点或者服务商不同，多地部署后端及其集群也不同。
### 优化手段
有些优化手段未必能让所有用户受益，如预加载往往只能让部分用户体验到明显的性能增长，来不及预加载的用户则享受不到。

如果优化效果未达到预期，优化手段是否生效本身也是一种维度。

### 浏览器
不同版本的浏览器数据聚合后再对比

### 操作系统
操作系统在移动端更加明显，比如安卓、IOS 客户端的API实现不同，网络库，图片库的策略不同。对比不同操作系统也有助于发现其中的性能问题。
