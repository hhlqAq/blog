# 第一环节：网络层优化
## 1. HTTP/2 全面启用

- 具体操作：通过Nginx、Caddy或云厂商CDN一键开启HTTP/2协议，关闭HTTP/1.1（保留降级兼容）；配置TLS1.2+，提升传输安全性与速度。
- 量化效果：多路复用解决HTTP/1.1队头阻塞问题，请求并发能力提升50%，页面加载速度提升20%-50%（请求数越多，收益越明显）。

## 2. 域名收敛与DNS优化

- 具体操作：① 域名收敛：将静态资源（JS、CSS、图片、字体）统一到1-2个域名下，避免使用5个以上域名（减少DNS解析次数）；② DNS预解析与预连接：在HTML头部添加预解析、预连接标签，提前建立DNS解析与TCP连接。
```js
<link rel="dns-prefetch" href="//cdn.xxx.com">
<link rel="preconnect" href="//cdn.xxx.com" crossorigin>
```
- 量化效果：DNS解析时间减少20-200ms/次，预连接可节省50-300ms连接耗时，域名收敛后请求并发更高效。
- 参考：域名越多，要建立的TCP/TLS连接就越多
  - 增加握手延迟
  - 破坏多路复用
  - 降低带宽利用率
  - 让优先级、推送等特性失效
  - 
## 3. 静态资源CDN全覆盖

- 具体操作：将所有静态资源（JS、CSS、图片、字体、视频封面）部署至主流CDN（阿里云、腾讯云、Cloudflare），配置CDN缓存规则、节点智能调度，确保用户就近访问；对大文件（如视频、大图片）启用CDN分片传输。
- 量化效果：静态资源加载速度提升30%-80%，跨地域访问延迟降低60%以上，核心资源加载成功率提升至99.9%。

## 4. 缓存策略优化（强缓存+协商缓存）

- 具体操作：① 静态资源（JS、CSS、图片、字体）：采用“强缓存+文件名hash”策略，设置Cache-Control: max-age=31536000（1年），文件名添加contenthash（如app.[contenthash].js），确保资源更新后能被及时缓存；② 接口请求：采用协商缓存（ETag/Last-Modified），设置Cache-Control: no-cache，避免重复请求，同时保证接口数据实时性；③ 缓存清理：提供版本更新触发缓存清理机制，避免旧缓存干扰。
- 量化效果：静态资源二次访问耗时降低90%以上，接口重复请求减少60%，带宽消耗降低40%。
  
# 第二环节：资源体积优化
核心目标：减少资源传输体积，缩短加载时间，降低带宽消耗，占整体优化收益的25%-35%。

## 1. JS/CSS优化
- 具体操作：
  - 压缩混淆：使用Terser压缩JS（去除注释、空格、变量名混淆），使用CSSNano压缩CSS，开启source-map（仅开发环境，生产环境关闭或隐藏）；
  - Tree-Shaking：开启webpack/rollup Tree-Shaking，剔除未使用的代码（需确保代码为ES模块，避免副作用）；
  - 代码分割：按路由、组件拆分代码，实现路由懒加载（React.lazy+Suspense、Vue异步组件），抽离公共依赖（如react、vue、axios），避免重复打包；
  - 冗余清理：删除未使用的依赖、冗余代码，替换大体积依赖（如用dayjs替代moment.js，体积减少80%）
- 量化效果：JS/CSS包体积减少xx%，首屏JS包体积控制在500KB以内（核心目标300KB），加载时间缩短xx%以上。
  
## 2.图片优化（C端核心优化点）
- 具体操作：
  - 格式优化：优先使用WebP/AVIF格式（体积比JPG/PNG小50%-80%），实现自动降级（不支持的浏览器回退至JPG/PNG）；

  - 懒加载：对首屏外的图片启用原生懒加载（loading="lazy"），配合IntersectionObserver实现自定义懒加载（兼容低版本浏览器）；

  - 响应式图片：使用srcset+size属性，根据设备分辨率加载对应尺寸图片，避免大图片在小屏幕上浪费带宽；

  - 小图处理：小于10KB的图片转Base64嵌入HTML（减少请求数），10-100KB的图片合并雪碧图（适用于图标类）；

  - 实时压缩：通过CDN实时压缩图片（调整质量、尺寸），根据场景设置合理压缩比（如列表图质量70%，详情图质量85%）

- 量化效果：图片体积减少50%-80%，图片加载时间缩短60%以上，CLS降低30%（避免图片加载导致的布局偏移）。

## 3. 字体优化（避免体验隐患）

- 具体操作：① 设置font-display: swap，避免字体加载期间出现FOIT（无文本闪烁）、FOUT（文本闪烁）；② 字体子集化：只保留常用汉字（如3000-5000个常用字），剔除生僻字，减少字体文件体积；③ 字体懒加载：非首屏字体延迟加载，首屏字体优先加载，配合CDN部署。
- 量化效果：字体文件体积减少60%-80%，避免字体加载导致的页面闪烁，CLS进一步降低，页面渲染更流畅。

# 第三环节：解析与渲染优化（提升首屏体验）

## 1. 关键CSS内联
- 具体操作：提取首屏必需的CSS（如导航、首屏Banner、核心内容样式），直接内联到HTML的`<style>`标签中，非首屏CSS（如详情页、评论区）异步加载（link rel="preload" + onload切换）。
- 量化效果：首屏渲染加速100-500ms，避免CSSOM阻塞HTML解析与渲染，白屏时间缩短xx%。
## 2. 非关键JS异步加载
- 具体操作：对非首屏必需的JS（如统计、埋点、广告、非核心组件JS），使用async(无依赖脚本)/defer(有依赖脚本)属性异步加载，避免阻塞HTML解析与CSS渲染；核心JS（如框架、路由核心）优先加载，可配合preload预加载。
- 量化效果：HTML解析时间缩短xxms，首屏渲染不受非核心JS影响，TTI提前xxms。
## 3. 预加载与预fetch优化
- 具体操作：① 预加载（preload）：对首屏核心资源（核心JS、关键CSS、首屏图片）进行预加载，确保优先加载；② 预fetch（prefetch）：对用户可能跳转的下一页面资源（如首页→详情页的JS/CSS）进行预加载，提升页面切换速度。
- 量化效果：核心资源加载时间缩短200-300ms，下一页面切换速度提升50%以上，实现“秒开”效果。
## 4. SSR/同构渲染（首屏直出核心方案）
- 具体操作：对首屏内容要求高的页面（如首页、详情页），落地SSR（服务端渲染）或同构渲染（Next.js、Nuxt.js、自研方案），服务端直接渲染首屏HTML，客户端接管后续交互，实现“首屏直出、后续SPA”。

# 第四环节：运行时优化（防卡顿，提升交互体验）
核心目标：减少主线程阻塞，解决卡顿、掉帧问题，提升交互响应速度，占整体优化收益的10%-15%。
## 1.长任务拆分与优化
- 具体操作：① 识别长任务：通过Chrome DevTools、Lighthouse识别超过50ms的长任务（主线程阻塞元凶）；② 拆分长任务：用requestIdleCallback、setTimeout将长任务拆分为多个短任务（每个任务≤50ms），避免阻塞主线程；③ 大计算迁移：将复杂计算（如数据筛选、格式化、图表渲染）迁移至Web Worker，不占用主线程资源。
- 量化效果：INP达标率提升xx%+，长任务占比从xx%降至x%以下，交互响应速度提升xx%，无明显卡顿。
## 2. 长列表优化
- 具体操作：对万条以上的长列表（如信息流、商品列表、评论区），使用虚拟列表（React：react-virtualized、react-window；Vue：vue-virtual-scroller），只渲染可视区域内的列表项，动态销毁不可视区域的DOM，减少DOM数量。
- 量化效果：DOM数量减少80%以上，长列表渲染时间缩短70%，滚动流畅度提升60%，无掉帧现象
## 3. 高频事件优化（防抖、节流）
- 具体操作：对scroll、resize、input、click等高频触发的事件，使用防抖（debounce）或节流（throttle），减少事件执行次数，避免主线程阻塞。
- 量化效果：高频事件执行次数减少80%以上，主线程压力降低，页面滚动、输入无卡顿。
## 4. 减少重排重绘（动画优化）
- 具体操作：① 动画优化：优先使用transform、opacity实现动画（不会触发重排，仅触发重绘），避免使用width、height、top、left等属性（触发重排）；② 批量DOM操作：通过DocumentFragment、离线DOM（display: none）批量修改DOM，减少重排次数；③ 避免读写样式交替：避免在循环中同时读取和修改DOM样式，减少强制同步布局。
- 量化效果：动画保持60fps（每秒60帧），无掉帧现象，重排次数减少70%，页面渲染更流畅。
# 第五环节：体验与稳定性优化
## 1. 布局偏移优化（降低CLS）
- 具体操作：① 图片/视频容器：设置固定宽高比（aspect-ratio）或固定宽高，预留内容空间，避免加载后布局偏移；② 动态内容：对异步加载的内容（如评论、推荐），提前预留占位符（骨架屏、灰色占位块）；③ 避免动态插入内容到页面顶部，如需插入，提前预留空间。
- 量化效果：CLS从0.3降至0.1以下（优秀标准），页面布局稳定，用户体验提升，跳出率降低10%。
## 2. 骨架屏与占位符优化
- 具体操作：首屏、异步加载区域（如详情页、评论区）使用骨架屏（模拟页面结构），图片用灰色占位块，接口请求时显示加载状态，避免用户看到空白页面产生焦虑。
- 量化效果：用户感知加载速度提升40%，跳出率降低15%，提升用户留存。
## 3. 异常处理与降级方案
- 具体操作：① 资源加载失败：对JS、CSS、图片等资源，设置加载失败降级方案（如图片加载失败显示默认图，JS加载失败提示用户刷新）；② 接口请求失败：实现接口重试机制（最多3次，间隔递增），失败后显示友好提示，提供重试按钮；③ 低版本浏览器：适配低版本浏览器（如IE11），提供基础功能降级，避免页面崩溃。
# 第六环节：工程化与监控体系
核心目标：实现优化方案可落地、可监控、可迭代，防止性能退化，确保优化效果长期稳定。
## 1. 工程化优化
- 具体操作：① 模块联邦/微前端拆包：按业务域拆分项目（如首页、详情页、个人中心），抽离公共依赖，实现模块独立开发、部署，单个页面包体积进一步下降30%；② 构建优化：使用webpack5（提升构建速度），开启缓存（babel缓存、webpack缓存），减少构建时间；③ 性能门禁：在CI/CD流程中添加性能门禁，检测包体积（如首屏JS≤500KB）、重复依赖、长任务，不达标则阻止部署；④ 规范约束：制定前端性能开发规范（如图片格式、缓存策略、代码分割要求），强制团队执行。
## 2. 性能监控体系
- 具体操作：① 监控工具：集成Sentry（异常监控）、阿里云ARMS/腾讯云前端监控（性能监控）、自研监控平台，实时采集核心性能指标（LCP、INP、CLS等）、异常数据、用户行为数据；② 告警设置：对指标不达标（如LCP>2.5s、INP>200ms）、异常率过高（如资源加载失败率>1%）设置分级告警（邮件、钉钉、企业微信），及时响应；③ 数据分析：定期分析监控数据，定位性能瓶颈（如某页面LCP不达标，排查是否为图片加载过慢），迭代优化方案。

function detectImageFormatSync() {
  const canvas = document.createElement('canvas');
  if (canvas.getContext && canvas.getContext('2d')) {
    // 检测 WebP
    const isWebpSupported = canvas.toDataURL('image/webp').indexOf('data:image/webp') === 0;
    if (isWebpSupported) return 'webp';
    
    // 检测 AVIF（部分浏览器不支持 canvas 导出 AVIF，仅做补充）
    try {
      const isAvifSupported = canvas.toDataURL('image/avif').indexOf('data:image/avif') === 0;
      if (isAvifSupported) return 'avif';
    } catch (e) {}
  }
  return 'jpg';
}

/**
 * 加载图片，自动选择最优格式
 * @param {string} baseUrl 图片基础 URL（不含后缀，如 '/images/banner'）
 * @param {string} fallbackExt 兜底后缀（如 'png'）
 * @returns {Promise<string>} 最终加载的图片 URL
 */
async function loadImageWithFallback(baseUrl, fallbackExt = 'jpg') {
  const format = await detectBestImageFormat();
  const extMap = {
    avif: 'avif',
    webp: 'webp',
    jpg: fallbackExt // 降级到传入的兜底格式
  };
  
  const finalUrl = `${baseUrl}.${extMap[format]}`;
  
  // 额外兜底：若最优格式图片加载失败，直接加载兜底格式
  return new Promise((resolve) => {
    const img = new Image();
    img.src = finalUrl;
    
    img.onload = () => resolve(finalUrl);
    img.onerror = () => {
      // 加载失败，降级到基础格式
      resolve(`${baseUrl}.${fallbackExt}`);
    };
  });
}

// 原生 JS 使用示例
async function renderImage() {
  const imgUrl = await loadImageWithFallback('/images/banner', 'png');
  const imgElement = document.createElement('img');
  imgElement.src = imgUrl;
  imgElement.alt = 'banner';
  document.body.appendChild(imgElement);
}

// React 组件中使用
function ImageWithFallback({ baseUrl, fallbackExt = 'jpg', alt }) {
  const [imgUrl, setImgUrl] = React.useState('');
  
  React.useEffect(() => {
    loadImageWithFallback(baseUrl, fallbackExt).then(setImgUrl);
  }, [baseUrl, fallbackExt]);
  
  if (!imgUrl) return <div className="img-loading">加载中</div>;
  return <img src={imgUrl} alt={alt} onError={() => setImgUrl(`${baseUrl}.${fallbackExt}`)} />;
}

// Vue 组件中使用

const props = defineProps(['baseUrl', 'fallbackExt', 'alt']);
const imgUrl = ref('');

  imgUrl.value = await loadImageWithFallback(props.baseUrl, props.fallbackExt);
});
</script>
