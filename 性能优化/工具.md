# DevTools
## Network面板
大致分为以下几个区域
- 选项
  - preserve log 在页面导航后，保留上一页面留下的网络请求
  - disable cache 暂时屏蔽浏览器的http缓存
- 网络请求列表
  - 每个请求都有一个对应的条，点击条可以查看该请求的详细信息
- 传输体积
  - 对应资源的传输体积，默认展示的是压缩后的体积，而非解码后的体积
- 时间 和 waterfall
  - 资源加载整体耗时，鼠标移动到waterfall上可以查看每个资源的加载耗时。
  - Queued at ：开始进入队列的时间，判断请求发起的时间是否符合预期，这个时候还没有真正发起请求
  - Started at ：请求真正发起的时间，判断请求是否符合预期，是否有阻塞其他资源的加载
  - queuing：以上两个时间相减就是队列时间，产生等待一般有以下一些原因
    - 资源优先级较低，如低优先级的图片等待css先加载完成。
    - 等待可用TCP请求的时间，如 HTTP/1 等待上一个请求完成复用TCP连接。
    - 由于浏览器并发过多进入等待状态，如同时发起了10个请求，而浏览器只允许同时发起6个请求，那么后面4个请求就会进入等待状态。
    - 从磁盘读取缓存的时间
  - stalled：被阻塞的时间，包含queuing的任意原因，还包括代理协商等时间。

## performance 面板
- 操作区：包含录制和清除。如果要分析页面的加载性能，直接点击刷新按钮，浏览器会自动重新加载页面并记录加载过程；如果要分析某个特定操作的性能，需要使用录制按钮手动录制。
- 选项：screenshots 录制过程中，是否截图； memory 录制过程中，是否记录内存使用情况。
- 概览：不同颜色代表当前CPU和网络占用情况，以及fps波动。
- 截图：录制过程中，会截取当前页面的截图，方便查看分析结果。
